# v0.2.0 Round 12 Stale Branch Governance Remediation

Status: fixed repo-side and GitHub-side

## Scope

Audit source: Round 12 stale branch attack surface report at `origin/master` around `9fa76004`.

Affected trust surface:

- public Git refs
- PR governance boundary
- stale-code regression surface
- reviewer and downstream install guidance

Protected principles:

- release provenance
- auditability
- no overclaim
- operational boundedness

## Repository Evidence

Before remediation, revalidation found:

- `delete_branch_on_merge=false`
- 65 total remote branches from the GitHub API
- 64 `codex/*` remote branches in local remote refs
- 47 `codex/*` branches fully merged into `origin/master`
- 17 `codex/*` branches with stale unmerged WIP commits and no open PR

After remediation:

- `delete_branch_on_merge=true`
- GitHub API branch inventory reports only `master`
- local remote refs after `git fetch --prune origin` report only `origin/master`

## Actions Taken

### B-048 â€” Stale unprotected branches with no cleanup policy

Status: fixed.

GitHub repository setting `delete_branch_on_merge` was enabled. Fully merged `codex/*` branches were deleted after verifying they were ancestors of `origin/master`.

A public branch policy was added at `.github/BRANCH_POLICY.md`.

A scheduled and manually runnable branch governance workflow was added at `.github/workflows/branch-governance.yml`.

### B-049 â€” Stale branches contain pre-fix vulnerable code

Status: fixed for public branch surface.

All stale public `codex/*` branches were deleted. The final branch inventory contains only `master`.

The deleted unmerged stale WIP branches had no open PRs and were hundreds of commits behind `origin/master`. Their branch names and unique commit subjects were captured during remediation in the operator transcript before deletion.

### B-050 â€” Public branch access without warning

Status: fixed.

`.github/BRANCH_POLICY.md` now states that `master` is the only public source of truth and that `codex/*` branches are ephemeral implementation branches, not release channels, install targets, support branches, or evidence sources.

## Controls Added

- `scripts/check/check-stale-branches.mjs` reports remote branch inventory and fails if merged `codex/*` branches remain.
- `.github/workflows/branch-governance.yml` runs weekly and on demand.
- The workflow permits temporary same-repository branches only while they are heads of open pull requests. It fails for any non-master branch without an active PR, for merged `codex/*` branches that remain, or if GitHub `delete_branch_on_merge` is disabled.

## Verification

Manual verification:

```bash
gh repo view AI-gateway-systems/attestor --json deleteBranchOnMerge --jq '.deleteBranchOnMerge'
gh api repos/AI-gateway-systems/attestor/branches --paginate --jq '[.[].name]'
git fetch --prune origin
git branch -r --format='%(refname:short)'
gh api --paginate "repos/AI-gateway-systems/attestor/pulls?state=open&per_page=100" \
  --jq '.[] | select(.head.repo.full_name == .base.repo.full_name) | .head.ref' \
  | sort -u > /tmp/open-pr-heads.txt
node scripts/check/check-stale-branches.mjs \
  --require-active-pr-only \
  --open-pr-heads-file /tmp/open-pr-heads.txt
```

Expected state:

- `deleteBranchOnMerge=true`
- persistent branch inventory: `["master"]`
- any temporary non-master branch exactly matches an open same-repository PR head
- no orphaned remote branch remains after a PR is merged or closed
- no `origin/codex/*` refs after prune

## No-Claims

This remediation does not prove production readiness, complete audit coverage, or absence of vulnerabilities on `master`.

This remediation does not recover or review abandoned WIP content. Stale WIP branch deletion is a governance cleanup, not feature acceptance.

This remediation does not prevent a future maintainer with write access from intentionally creating a branch. It adds policy, cleanup evidence, and scheduled detection.
