# Attestor v0.1.0-evaluation Release Notes

**Release type:** GitHub pre-release / evaluation baseline

**Tag:** `v0.1.0-evaluation`

**Package version:** `0.1.0-evaluation`

**Primary reviewer path:** [Attestor Evaluation Packet v0.1](v0.1-evaluation-packet.md)

This release packages the current Attestor repository as a reproducible local evaluation proof. It is meant to let an outside engineer run the core admission path, inspect proof artifacts, and verify the local gate before deeper architecture review.

## What To Run

Start from a clean checkout at the release tag:

```bash
npm ci
npm run example:admission
npm run example:customer-gate
npm run proof:surface
```

For the portable proof-showcase path without external API credentials:

```bash
npm run showcase:proof
npm run verify:cert -- .attestor/showcase/latest/evidence/kit.json
```

`verify:cert` reports `PROOF_DEGRADED` and exits non-zero when the proof is local PostgreSQL-backed but upstream-model fixture-backed. That is expected for this evaluation release; use `npm run verify` as the green release gate.

Before treating the release tag as reviewable:

```bash
npm run verify
```

## What This Evaluation Release Demonstrates

- A concrete finance wedge: AI-assisted output -> structured financial record release.
- A bounded decision vocabulary: `admit`, `narrow`, `review`, `block`.
- A short local admission path with one admitted consequence and one blocked fail-closed consequence.
- A customer-side gate that proceeds only when Attestor returns an allowed decision.
- Local proof-surface rendering under `.attestor/proof-surface/latest/`.
- Portable proof-showcase output under `.attestor/showcase/latest/` without external API credentials.
- Independent kit inspection through `npm run verify:cert -- .attestor/showcase/latest/evidence/kit.json`, with expected `PROOF_DEGRADED` posture unless a live upstream model is configured.
- A full local verification gate through `npm run verify`.
- Package-script and TypeScript hygiene hardening: `npm test` and `npm run verify` now run through `scripts/run/run-suite.mjs`, and unused-code checks are part of the normal TypeScript gate.

## What Changed Since The Initial Public Repo Shape

- Added a runnable evaluation packet for outside reviewers.
- Hardened the reviewer path with 15-minute and 45-minute flows.
- Added an artifact map for terminal output, proof-surface output, showcase packets, and verification kits.
- Clarified that package `exports` are repository, self-hosted, and commercial/customer-operated import boundaries, not a public npm availability claim.
- Changed the package version from `1.0.0` to `0.1.0-evaluation` so the repository no longer implies stable public-package maturity.
- Added tests that protect the evaluation packet, release-note boundary, package script runner, package privacy posture, and command existence.
