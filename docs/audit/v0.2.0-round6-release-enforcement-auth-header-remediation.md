## Re-Audit Notes

The fallback `attestor-release-token` header is not removed in this slice. It
is treated as an intentional alternate presentation path, but it no longer
accepts folded, comma-joined, parameterized, or whitespace-bearing credentials.

Header normalization helpers may still join duplicate values for non-secret
binding headers because those headers are used for canonical request binding.
The new parser rejects joined authorization credentials at the PEP boundary.
