# v0.2.0 Round 11 Postgres SQL Governance Remediation

Status: fixed repo-side

## Scope

Audit source: Round 11 Postgres connector review at `origin/master` around `cb4b9159`.

Affected trust surface:

- Postgres proof connector SQL preflight
- Predictive guardrail SQL governance
- Snowflake connector reuse of the shared SQL governance helper

Protected principles:

- fail-closed boundary
- data minimization and redaction
- operational boundedness

## Findings

### B-044 — Dynamic import via `Function()`

Status: fixed for `src/connectors/postgres.ts`.

The Postgres connector no longer uses `Function('return import("pg")')` inside the connector module. It now uses an expression-based dynamic `import(moduleName)` helper, preserving optional runtime loading without an implied-eval pattern in the connector.

Limitation: other Postgres-backed service/runtime modules still use the same legacy dynamic import pattern and are out of scope for this narrow connector remediation.

### B-045 — Incomplete SQL comment stripping

Status: fixed.

`sqlForGovernance()` now strips PostgreSQL nested block comments with a scanner instead of a non-nested regex. It continues to strip string and dollar-quoted string contents before governance pattern matching so blocked keywords inside data literals do not create false positives.

### B-046 — Missing Postgres operational commands in the preflight denylist

Status: fixed.

The preflight denylist now includes additional PostgreSQL operational commands that are not part of the bounded read-only connector contract, including `COPY`, `LOCK`, `VACUUM`, `ANALYZE`, `REINDEX`, `CLUSTER`, `COMMENT`, `LISTEN`, `NOTIFY`, `UNLISTEN`, `DISCARD`, `REFRESH MATERIALIZED VIEW`, and `SECURITY LABEL`.

This remains defense-in-depth. Runtime execution still opens `BEGIN TRANSACTION READ ONLY`; PostgreSQL documents that read-only transactions disallow broad write classes but do not prevent every possible disk write. The connector therefore keeps both app-level preflight and DB-level read-only enforcement.

## External anchors

- PostgreSQL `SET TRANSACTION` documentation: read-only mode and its limits: <https://www.postgresql.org/docs/current/sql-set-transaction.html>
- PostgreSQL `COPY` documentation: server-side file/program capability and privilege implications: <https://www.postgresql.org/docs/current/sql-copy.html>
- PostgreSQL `LOCK`, `VACUUM`, `REINDEX`, and `CLUSTER` command documentation: operational commands outside the bounded read-only query contract: <https://www.postgresql.org/docs/current/sql-lock.html>, <https://www.postgresql.org/docs/current/sql-vacuum.html>, <https://www.postgresql.org/docs/current/sql-reindex.html>, <https://www.postgresql.org/docs/current/sql-cluster.html>
- PostgreSQL `LISTEN` and `NOTIFY` documentation: asynchronous notification commands outside the read-only proof query contract: <https://www.postgresql.org/docs/current/sql-listen.html>, <https://www.postgresql.org/docs/current/sql-notify.html>
