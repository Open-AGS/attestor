## Findings

### B-051 - Audit lifecycle has doc enforcement but no state-machine enforcement

Status: fixed for repository-side evidence checks.

Repository evidence:

- `scripts/check-audit-finding-evidence.mjs` validates the canonical lifecycle/template contract.
- The checker scans closed lifecycle records and fails when required closure fields are missing.
- The checker validates `fixed` rows in the historical tracker have at least PR, audit doc, or test evidence.
- `test:audit-finding-evidence` exposes the checker as a package script.
- `tests/audit-finding-lifecycle.test.ts` executes the checker.

Smallest safe fix:

Add a machine-readable evidence checker without rewriting historical audit findings or claiming external audit closure.

### B-052 - Findings are not consistently mapped to CWE/NIST/OWASP in tracker/remediation evidence

Status: fixed for canonical lifecycle/template and future closed records.

Repository evidence:

- The canonical remediation evidence schema now includes `Mapping:`.
- The compact audit ledger template now includes `Mapping:`.
- The checker requires a closed lifecycle record's `Mapping:` line to cite at least one accepted taxonomy/source family: CWE, NIST, OWASP, STRIDE, STPA, ASVS, or DSOVS.

Smallest safe fix:

Add mapping as a required closure field for future closed evidence records. For historical tracker rows, add only the minimum existing audit-doc and test references needed to make fixed-status evidence machine-checkable; do not convert old rows into full lifecycle records without original closure evidence.

## Mapping

- CWE-20: improper input validation, for machine-checking evidence fields before accepting closure.
- NIST SP 800-115: technical assessment reporting and mitigation evidence discipline.
- NIST CSF 2.0: risk documentation, categorization, and governance framing.
- OWASP ASVS / DSOVS: verifiable requirements and DevSecOps issue tracking/test-management anchors.

## No-Claims

This remediation does not certify Attestor, does not complete the full enterprise audit, and does not state that every historical remediation note has been normalized into the new schema.

This remediation enforces the canonical lifecycle contract for closed records and the historical tracker's minimum evidence posture. It is not a substitute for external audit review.
