## Re-Audit Notes

The original report described a broad fail-open surface. Repository evidence
narrows the validated issue: break-glass is opt-in, distinguishable from normal
acceptance, and cannot bypass invalid signatures. The remaining accepted gap is
boundedness: a misconfigured long-lived grant should not be treated as active
by the webhook receiver.

This fix aligns the webhook receiver with the degraded-mode control default
TTL. It does not add tenant-level rate limits or post-incident review
automation; those remain broader operational controls.
